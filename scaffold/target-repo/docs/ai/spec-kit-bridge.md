## Ownership Boundary
- Treat official Spec Kit files as Spec Kit-owned: `.specify/`, `specs/`, and official generated prompt, agent, script, template, or skill files.
- Treat the managed Spec Kit bridge summary inside [AGENTS.md](../../AGENTS.md) as enhancer-owned and safe for future bridge refreshes.
- Treat this file and any installed bridge skills as enhancer-managed guidance that may be regenerated by later enhancer refreshes.
- Treat enhancer-owned bridge skills as `.codex/skills/` files only. If `.agents/skills/` exists, treat it as an external compatibility surface; do not mirror, migrate, or overwrite files there.

## Working Rule
- When a feature already has Spec Kit artifacts, read the active spec and plan before implementing code.
- Keep implementation, validation, and review notes aligned with the current feature artifacts instead of only the latest prompt.
- Do not use enhancer flows to rewrite official Spec Kit templates or feature history.
